Linux 启动过程

1、BIOS开机自检，按照BIOS中设置的启动设备（通常是硬盘）来启动
2、操作系统接管硬件以后，首先读入 /boot 目录下的内核文件。
3、运行init()，init 进程是系统所有进程的起点，init 程序首先是需要读取配置文件 /etc/inittab。
4、许多程序需要开机启动。它们在Windows叫做”服务”（service），在Linux就叫做”守护进程”（daemon）。init进程的一大任务，就是去运行这些开机启动的程序。启动时根据运行级别确定运行的程序，运行级别3：完全的多用户状态(有NFS)，登陆后进入控制台命令行模式，运行级别5：X11控制台，登陆后进入图形GUI模式。
5、在init的配置文件中有这么一行： si::sysinit:/etc/rc.d/rc.sysinit　它调用执行了/etc/rc.d/rc.sysinit，做系统初始化
6、用户登录系统，一般来说，用户的登录方式有三种：（1）命令行登录（2）ssh登录（3）图形界面登录
总结：
磁盘启动—>操作系统接管—>读入内核文件—>init进程—>守护进程（根据优先级选择）—>系统变量初始化—>用户系统登陆

二叉树最低公共祖先

Question:给出二叉树与两个节点，找到该二叉树中两个节点最早的公共祖先

Solution:
TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
if (!root) return NULL;
if (root == p || root == q) return root;
TreeNode L = lowestCommonAncestor(root->left, p, q);
TreeNode R = lowestCommonAncestor(root->right, p, q);
if (L && R) return root; // if p and q are on both sides
return L ? L : R;
}

Explain:
本题的难点在于递归的时候状态的传递，如何告知其他子节点父节点已经找到了一个还是0个目标节点，如果找到了1个那么这个节点到底是第一个节点还是第二个节点

//type 0:nothing found 1:found p 2:found q
TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q,int type){
if (!root) return NULL;
switch(type){
case 0:
if(root == p)
return lowestCommonAncestor(root->left, p, q, 1);
else if(root == q)
return lowestCommonAncestor(root->left, p, q, 2);
else{
TreeNode left = lowestCommonAncestor(root->left, p, q, 1);
TreeNode right = lowestCommonAncestor(root->left, p, q, 1);
if (left) return left;
if (right) return right;
else return NULL;
}
break;
case 1:
if(root == q) return root;
else {
TreeNode left = lowestCommonAncestor(root->left, p, q, 1);
TreeNode right = lowestCommonAncestor(root->left, p, q, 1);
return left?left:right;
}
break;
case 2:
if(root == p) return root;
else {
TreeNode left = lowestCommonAncestor(root->left, p, q, 2);
TreeNode right = lowestCommonAncestor(root->left, p, q, 2);
return left?left:right;
}
}
}

这个是我的做法，但是没有第一种方法好：
return 表示该节点为根的子树至少有一个目标节点
如果左右递归均不为空，则表示左右节点都有目标节点，则返回这个节点
如果左边为空，则表示不在左子树上，在右子树或者其他兄弟数上，返回右子树

tips:
写递归函数的时候，多种情况从终结的情况写起，往往终结的情况比较简单易分析,这对写其他复杂的情况有帮助
改变return的条件可能可以得到不一样的效果

模糊效果时遇到的困难

在做电影界面优化的时候有一个地方要求用模糊效果处理，效果是这样的

使用下面这段代码可以实现模糊效果：

-webkit-filter: blur(20)

查了一下这句话的实现原理，是把每个点向周围20个像素进行发散，所以在图片原来大小的周围也会出现一些淡淡的模糊效果，要想图片维持原来的大小，必须加上

overflow： hidden

做完这些后发现在电脑上达到了预期的效果。

在手机上进行测试后发现有的手机上并没有出现预期的模糊效果，原来是有的手机并不支持这个效果，通过：

if('-webkit-filter' in document.element.style)

来检查是否支持滤镜属性
与美工商量后，决定对于不支持这个属性的手机做出如下处理：
将图片放大去中间的一部分，然后加上一片黑色的透明度50%的层
对于支持模糊效果的手机，考虑到部分图片是白色的，由于字也是白色的，所以在图片上也会加上不透明度为80%的黑色层

然后在手机上再进行测试，发现在三星的手机上模糊效果非常的卡，非常影响用户的体验，没办法还是改成了都不支持的效果，如果要想使用这个效果，并且获得比较好的用户体验，应该在数据库中存一张模糊的图品，然后前端只需要获取该图片当背静就可以了，减少了渲染的时间。
