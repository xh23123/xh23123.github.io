一行中并排显示

下面这个类使子元素水平存在，并且中间是在同一条水平线上的
.box {
display: -webkit-box;
-webkit-box-pack: start;
-webkit-box-orient: horizontal;
-webkit-box-align: stretch;
}
box-pack 属性规定当框大于子元素的尺寸，在何处放置子元素
box-orient 属性规定框的子元素应该被水平或垂直排列。

如果不想让子元素处在同一条水平线上，只需要
display: -webkit-box;
就可以了

模糊效果时遇到的困难

在做电影界面优化的时候有一个地方要求用模糊效果处理，效果是这样的

使用下面这段代码可以实现模糊效果：

-webkit-filter: blur(20)

查了一下这句话的实现原理，是把每个点向周围20个像素进行发散，所以在图片原来大小的周围也会出现一些淡淡的模糊效果，要想图片维持原来的大小，必须加上

overflow： hidden

做完这些后发现在电脑上达到了预期的效果。

在手机上进行测试后发现有的手机上并没有出现预期的模糊效果，原来是有的手机并不支持这个效果，通过：

if('-webkit-filter' in document.element.style)

来检查是否支持滤镜属性
与美工商量后，决定对于不支持这个属性的手机做出如下处理：
将图片放大去中间的一部分，然后加上一片黑色的透明度50%的层
对于支持模糊效果的手机，考虑到部分图片是白色的，由于字也是白色的，所以在图片上也会加上不透明度为80%的黑色层

然后在手机上再进行测试，发现在三星的手机上模糊效果非常的卡，非常影响用户的体验，没办法还是改成了都不支持的效果，如果要想使用这个效果，并且获得比较好的用户体验，应该在数据库中存一张模糊的图品，然后前端只需要获取该图片当背静就可以了，减少了渲染的时间。

到底部时自动加载更多

判断页面到底了进行判断
clientHeight div的显示高度
offsetHeight 总高度
scrollTop 上边的位置 [0,clientHeight-offsetHeight]
1、判断滚动条滚动到最底端： scrollTop == (offsetHeight – clientHeight)
2、在滚动条距离底端50px以内： (offsetHeight – clientHeight) – scrollTop <= 50
3、在滚动条距离底端5%以内： scrollTop / (offsetHeight – clientHeight) >= 0.95

关于切图

一般前端会给两张图，一张是png格式的图，一张是jpg格式的图，png格式就是我们要用的图，jpg格式上面一般会标有每个东西的左上角坐标，以及图形的长和宽

我们要使用应该有一下2个css类

源类，就是整张图片，大小应该是整张图片的大小
.movie-star-ele {
background: url(‘/res/image/movie/movie_star.png’)no-repeat;
background-size: 9.2rem 15.35rem;
}

该类标出这个图形的坐标，大小
.movie-star-five {
background-position: 0 -2.9rem;
width: 6.7rem;
height: 1.2rem;
}

这样就可以显示出想要的图片了

规定行数与查看更多的实现

要规定几行，就用下面这个类就可以，就是ellipsis表示超过了就是…的表示
.three-line {
display: -webkit-box;
overflow: hidden;
text-overflow: ellipsis;
-webkit-box-orient: vertical;
-webkit-line-clamp: 3;
}
加上这个类后就可以使他只有规定的行然
如果想实现点击查看更多后显示完整的内容，只需要在点击事件中把这个类中去掉即可

Hexo更换皮肤

今天晚上很开心，没有加班，早早的就回来了，现在觉得公司的福利真的非常不错啊，早餐+晚餐+水果+按摩==

开始说正事，今天觉得这个皮肤实在是太丑了，就换了一下皮肤。

第一步：安装
$ git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
前面是路径，后面是存放的路径

第二步：配置
修改hexo根目录下的 _config.yml ： theme: **

第三步：更新
cd themes/yilia
git pull

然后再更新一下就好噜！
